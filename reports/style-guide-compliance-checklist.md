# Style Guide Compliance Checklist

## Tone & Style
- Uniform brand voice across tool pages and guides
- Friendly, clear language; short paragraphs; active voice
- Consistent terminology: Tool, Feature, Input, Output, Results, Settings

## Readability & Structure
- H1 used for page title; H2/H3 for sections
- Scannable formatting with bullets and short lists
- Clear flow from purpose → how → output → benefits → related

## SEO
- Primary and secondary keywords present in metadata
- Canonical tags set for each route
- Structured data: `SoftwareApplication`, `BreadcrumbList`, `Article`, `FAQPage`
- Internal links to related tools and guides

## Accessibility
- Descriptive link text and aria labels on interactive elements
- Color contrast via design tokens
- Keyboard-friendly focusable links and buttons

## Technical
- Mobile-responsive layout and typography
- Client-side tool runners only; no blocking external calls
- Valid JSON-LD payloads with connected `@graph`
